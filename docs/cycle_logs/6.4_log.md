## Cycle 1 — 2026-05-21T20:46:50+09:00

### Intent
Implement the Meta Upgrade & Lobby UI system for POTOP. This includes:
- `GemWallet.cs`: Persistent Gem currency manager (earn/spend, PlayerPrefs save).
- `MetaUpgradeManager.cs`: Manages 6 permanent upgrade slots, reads from `MetaUpgradeData` SOs, applies effects.
- `MetaUpgradeData.asset` ×6 (ScriptableObject definitions): one per upgrade type.
- `LobbyScreen.uxml/uss`: Lobby screen root layout using UI Toolkit.
- `MetaUpgradeCard.uxml`: Individual upgrade card template.
- `LobbyController.cs`: Wires UI to logic via EventBroker; handles card click → gem spend → level increment.

### Analysis
- Project uses static `EventBroker` (pub/sub, K&R style, `Potop.Client.Core.Events` namespace).
- All UI must use UI Toolkit (UXML/USS). Canvas/UGUI prohibited.
- C# style: K&R braces, explicit access modifiers, `[SerializeField] private Type _fieldName`.
- Naming: PascalCase classes/methods, _camelCase private fields, UPPER_SNAKE_CASE constants.
- Existing UI references: GameHUD uses `UIDocument`, queries elements by name, subscribes to EventBroker in OnEnable/OnDisable.
- `UpgradeSelectController.cs` is the closest pattern for LobbyController — card building in code.
- 6 upgrade types from GDD: 강화 외장(HP), 고속 모터(Speed), 에너지 집적(SkillCharge), 자력 코어(GemRadius), 냉각 시스템(Overcharge), 정밀 스캐너(HazardRate).
- Cost doubles per level (500→1000→2000...), max level to set conservatively at 5.
- Save via PlayerPrefs (temporary); keys per upgrade ID.
- LobbyController decoupled from game-play logic via EventBroker events.
- New events needed: `MetaUpgradePurchasedEvent`, `GemChangedEvent`.
- The `Assets/Data/Meta/` directory needs to be created for ScriptableObject assets.
- UI assets go in `Assets/UI/Lobby/`.

### Failure Modes
1. `MetaUpgradeData` SO assets not assigned in Unity inspector → `NullReferenceException` in MetaUpgradeManager at startup. Mitigation: null-check each slot with a warning log.
2. PlayerPrefs key collision if two upgrades share a prefix. Mitigation: use fully qualified key format `"meta_upgrade_{upgradeId}_level"`.
