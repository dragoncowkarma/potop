## Cycle 1 — 2024-05-21

### Intent
Implement the Item Drop system for the POTOP project, specifically including a Repair Kit, Magnet, and Smart Bomb, alongside a wave-integrated probability-based ItemSpawner.

### Analysis
The requirement asks for three item types: Repair Kit, Magnet, Smart Bomb. We must use `PoolManager` and `EventBroker`. Existing mechanics rely on `Player` tag.

### Plan
1. `ItemDropData`: Create a ScriptableObject defining item stats.
2. `ItemDrop`: Base class with object pooling despawn.
3. `ItemSpawner`: Subscribes to events, spawns items using weighted logic, handles collection events for actual item effect execution.

### Failure Modes
1. `NullReferenceException` if PoolManager or GameManager instances are missing during tests or gameplay.
2. Smart Bomb overlap sphere might miss enemies if they lack colliders or layer mask is incorrect.
