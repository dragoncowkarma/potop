## Cycle 6.1 — 2023-10-27

### Intent
Implement the `OverchargeController.cs` for the Overcharge System as specified in task 6.1, including gauge logic (charge, overheat, decay), event broadcasting (`OnOverchargeStateChanged`), and SO-based configuration (`OverchargeData`).

### Analysis
The task requires:
1. Creating `OverchargeController.cs` in `Assets/Scripts/Gameplay/Combat/`.
2. Managing a gauge (0~100) using Unity Input System (`started`/`canceled`).
3. Transitioning between states: Idle, Active (charging), Overheat.
4. Using ScriptableObject (`OverchargeData`) instead of magic numbers.
5. Broadcasting state changes using `EventBroker`.
The file needs a `.meta` file. The events need to be scoped locally to avoid modifying `GameEvents.cs`.

### Plan
1. Create `OverchargeController.cs` with `OverchargeState` enum, `OverchargeStateChangedEvent` struct, `OverchargeData` class, and `OverchargeController` MonoBehaviour.
2. Bind input action safely and manage gauge according to task rules (+20/s active, -15/s idle, 3s overheat).
3. Ensure no magic numbers in code by mapping defaults in `OverchargeData`.
4. Add `.meta` file.

### Failure Modes
1. NullReferenceException if `_overchargeData` or `_overchargeAction` is not assigned in the inspector.
2. `EventBroker` not correctly imported or event struct missing.
3. Overheat duration not resetting the gauge correctly.
